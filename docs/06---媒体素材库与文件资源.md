# 媒体素材库与文件资源

最近更新：2026-06-08

## 功能定位

媒体素材库管理上传素材、生成图片、生成视频、音频和可复用文件引用。媒体文件不进入 PostgreSQL，数据库只保存元数据和相对路径。

## 当前项目情况

- 媒体素材库入口是 `/media-library`。
- 当前页面支持全部/图片/视频筛选、关键词搜索、分页、上传进度、预览、选择和批量删除。
- 素材主表是 `assets`，保存名称、类型、分类、URL、本地路径、大小、MIME、宽高、时长、来源生成记录和创建人。
- 上传和生成文件统一应落在 `data/storage`，PostgreSQL 只保存引用。
- 普通用户只能删除自己创建的素材，管理员不受此限制。

## 当前已落地能力

- 支持图片、视频等素材列表。
- 支持上传素材、预览素材、删除素材。
- 支持素材创建人记录。
- 支持从图片生成结果或视频生成结果导入素材。
- 服务器持久化目录为 `/opt/ai-video/qnvideo-gzt/data`，项目内媒体统一使用 `data/storage`。

## 当前问题与不足

- 素材库与角色/场景/道具库之间的边界需要更明确。
- 删除素材时，对已有项目、分镜、生成记录的引用影响需要更可见。
- 文件清理策略需要兼顾节省空间和避免误删。

## 方案池 / 待确认讨论

- 是否把素材分为原始上传、生成结果、精选素材、项目引用四类。
- 是否增加“被哪些项目/分镜引用”的反查。
- 是否增加素材归档而不是直接删除。

- 梳理删除素材时的引用保护策略。
- 增加素材列表筛选：创建人、来源、类型、项目引用、生成任务。

- 媒体文件不写入 PostgreSQL。
- 删除素材不会误删仍被项目引用的关键文件。
- 普通用户只能编辑或删除自己创建的素材。
