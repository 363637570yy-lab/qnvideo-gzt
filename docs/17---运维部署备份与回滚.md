# 运维部署备份与回滚

最近更新：2026-06-09

## 功能定位

运维部署备份与回滚规范用于保证 qnvideo-gzt 在服务器上的更新可观察、可验证、可回滚，避免误删持久化数据、误覆盖配置、黑盒执行脚本或带病上线。

## 部署边界

- 本地项目目录：`D:\BCKF\01---短视频服务器\qnvideo-gzt`
- 服务器目录：`/opt/ai-video/qnvideo-gzt`
- Web 端口：`20880`
- API 健康检查：`127.0.0.1:20881/health`
- 容器：`qnvideo-gzt-web`、`qnvideo-gzt-backend`
- 持久化目录：`/opt/ai-video/qnvideo-gzt/data`

## 目标目录

```text
deploy/

  diagnostics/              本地和服务器诊断脚本
  maintenance/              数据修复、清理、巡检脚本
  migrations/               一次性迁移辅助脚本
```

这些目录是目标归属，只有需要脚本或部署配置时才创建；不要为了空目录提交无内容文件。

## 部署前检查

本地必须确认：

```powershell
git status --short --branch
git remote -v
```

服务器必须确认：

```bash
cd /opt/ai-video/qnvideo-gzt
git status --short --branch
git remote -v
docker compose -f docker-compose.codex.yml ps
```

如果服务器有未提交修改，不直接覆盖。`data/` 是持久化目录，绝不能被清理、reset 或重新初始化。

## 标准部署流程

```bash
cd /opt/ai-video/qnvideo-gzt
git fetch origin main
git pull --ff-only origin main
docker compose -f docker-compose.codex.yml build backend
docker compose -f docker-compose.codex.yml build web
docker compose -f docker-compose.codex.yml up -d
```

部署后必须验证：

```bash
docker compose -f docker-compose.codex.yml ps
curl -sS -o /dev/null -w 'health=%{http_code} %{time_total}\n' http://127.0.0.1:20881/health
curl -sS -o /dev/null -w 'web_local=%{http_code} %{time_total}\n' http://127.0.0.1:20880/
curl -sS -o /dev/null -w 'web_public=%{http_code} %{time_total}\n' http://www.qnzn.top:20880/
```

## 备份策略

需要备份的内容：

- PostgreSQL 数据库。
- `/opt/ai-video/qnvideo-gzt/data` 持久化文件。
- `.env` 和部署配置，不进入公开仓库。
- 关键版本号和当前 Git commit。

备份要求：

- 备份命名包含项目、时间戳和类型。
- 数据库备份和文件备份要能对应同一时间点。
- 恢复步骤必须先在非破坏路径验证。
- 备份日志不得泄漏数据库密码、API Key、代理订阅。

## 回滚策略

代码回滚优先：

```bash
git log --oneline -5
git switch main
git pull --ff-only origin main
```

如需回滚到旧 commit，必须先确认：

- 数据库迁移是否可兼容旧代码。
- 新生成文件是否会被旧代码误读。
- 前端构建产物是否需要重建。
- 当前容器状态和日志是否已保存。

禁止：

- 无确认执行 `git reset --hard`。
- 删除 `data/`。
- 覆盖 `.env`。
- 对服务器执行来源不明的一键脚本。

## 真实浏览器验收

涉及页面、登录态、生成链路、资源展示、AI 配置时，部署后应做真实浏览器检查：

- 页面能打开。
- 登录页或制作页关键文本存在。
- API health 正常。
- 静态文件 `/static/...` 可访问。
- 控制台无明显首屏错误。
- 生成链路至少完成到任务创建或配置测试。
