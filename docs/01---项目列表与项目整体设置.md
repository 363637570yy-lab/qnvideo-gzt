# 项目列表与项目整体设置

最近更新：2026-06-10

## 功能定位

项目是所有剧本、角色、场景、道具、分镜、图片、视频和合成结果的业务容器。项目整体设置负责给后续生成链路提供默认规格。

## 当前项目情况

- 项目列表入口是 `/`，项目详情入口是 `/drama/:id`，制作入口是 `/film/:id`。
- 项目结构由 `dramas` 和 `episodes` 承载，项目下挂角色、场景、道具、分镜、图片、视频和合成记录。
- 项目详情页已展示剧集信息、分集列表、剧内资源库和制作资源。
- 制作页已能保存故事梗概、分集剧本、项目进度和部分媒体规格。
- 当前应把画幅、画风、默认语言、默认片段时长继续收敛为项目整体设置。

## 当前已落地能力

- 项目列表页展示用户可访问的项目。
- 支持创建、导入、导出、删除项目。
- 项目支持标题、简介、类型、风格、总集数等信息。
- 项目整体设置已明确应包含画幅、主画风、默认语言、默认每段时长。
- 制作页项目整体设置状态已收敛到 `useProjectSettings.js`，包括画幅、主画风、默认语言、默认时长、图像规格、视频规格和延迟保存。
- 制作页项目整体设置已在所有工作台内容顶部显示，但不做滚动吸顶；页面向下滚动时控制区随内容正常滚走。
- 制作页项目整体设置改为紧凑列表式布局：`项目整体设置` 标题与 `项目画幅` 同排，所有设置项采用“上方名称、下方筛选框/按钮”的两行结构；保存成功后通过 toast 提示“修改已实时保存，再次生成素材、分镜或视频时将按当前设置生效。”，避免用户误以为已生成内容会自动重算。
- 制作页项目整体设置初始化、切换项目或刷新摘要时不触发保存成功 toast；只有用户实际修改设置并保存成功后才提示。
- 制作页顶部已恢复本次生成模型选择框，文本、图片、视频、音频可按“自动 / 指定配置”选择；左侧导航不再放模型下拉选择，控制区不再提供管理员配置按钮。
- 制作页左侧导航与顶部工作台 tab 对齐为 `故事剧本`、`角色`、`场景`、`道具`、`分镜`、`视频/合成` 六项；阶段数量和完成态优先读取工作台摘要接口 `progress_steps`，不再依赖懒加载后的本地数组推断。
- 工作台摘要接口新增当前集 `storyboard_outline` 轻量分镜目录，左侧分镜列表无需先打开分镜 tab 即可展示；点击分镜目录项时才加载完整分镜工作台并跳转到对应卡片。
- 剧集管理页前端路由是懒加载，页面数据也应优先走 `GET /api/v1/dramas/:id/basic` 轻量详情，只取项目基础信息、分集和分镜数量；本剧制作资源按角色、场景、道具 tab 再分别加载。
- 普通用户只能访问自己的项目，管理员可查看全部项目。

## 当前问题与不足

- 项目整体设置和单次分镜生成设置容易混在一起。
- 项目整体设置、管理员 AI 配置、本次生成模型策略三者已经初步视觉隔离；后续仍需把“本次指定模型”的入口和权限口径继续产品化。
- 制作页已开始改为“轻量摘要首屏 + 标签页懒加载”：首屏使用工作台摘要接口，剧本、角色、场景、道具、分镜、视频/合成按标签页加载；后续还要继续拆组件并减少旧 `loadDrama()` 兼容刷新。
- 项目创建时没有足够强的创作规格确认，导致后续角色、场景、分镜图、视频生成容易各自为政。
- 项目列表与制作链路之间缺少“当前项目完成度 / 风险提示 / 下一步动作”的引导。

## 方案池 / 待确认讨论

- 项目创建流程是否拆成“基础信息 + 视觉规格 + 生成策略”三步。
- 项目卡片是否显示剧本、资产、分镜、图片、视频、合成状态。
- 项目整体设置是否应支持锁定，锁定后批量生成必须使用同一画幅和主画风。
- 制作页各工作台内容顶部固定保留“项目整体设置 + 一键动作 + 本次生成模型策略”三行结构，但不做滚动吸顶：
  - 项目整体设置：画幅、图像规格、视频清晰度、默认时长、默认语言、主画风。
  - 一键动作：一键生成素材及分镜文本、一键成片（测试中慎用）。
  - 本次生成模型策略：文本、图像、视频、音频使用选择框展示“自动 / 指定配置”，并保留刷新入口；管理员 AI 配置仍只从顶部栏或管理入口进入。
- 本次生成模型策略只影响当前制作页后续生成动作的路由覆盖，默认自动；它不替代管理员 AI 配置，也不应写回全局模型配置。
- 本次生成模型策略以轻量选择框展示：`文本 自动/指定`、`图像 自动/指定`、`视频 自动/指定`、`音频 自动/指定`。未配置时下拉无可用项，生成失败仍由后端错误提示和任务诊断承接。
- 制作页首屏应优先调用 `GET /api/v1/workbench/projects/:drama_id/summary`，只拿项目整体设置、本次模型策略摘要、一键动作状态和各标签计数/任务数；具体剧本、资源、分镜、视频数据在用户打开对应标签页后再加载。
- 工作台摘要接口必须提供 `progress_steps`：后端仍可按剧本、角色、道具、场景、分镜文本、分镜图、视频进度输出细粒度步骤，前端左侧导航合并展示为 `故事剧本`、`角色`、`场景`、`道具`、`分镜`、`视频/合成` 六项；请求可携带 `episode_id` 来返回当前集范围的剧本、分镜和视频进度。
- 工作台摘要接口必须提供 `storyboard_outline` 轻量目录，只包含分镜 id、序号、标题、段落信息和更新时间，不返回图片、视频或媒体历史。

## 验收清单

- 新建项目后，画幅、画风、语言、默认时长能被后续角色、场景、分镜和视频生成读取。
- 单次生成参数不会覆盖项目整体设置，除非用户明确保存为项目默认值。
- 本次生成模型策略显示的文本、图像、视频、音频路由与实际请求传参一致，切换后可在模型调用日志或任务诊断中追踪。
- 打开制作页时，首屏不请求全量分镜和资源历史；只有进入对应标签页后才加载该标签页完整数据。
- 打开剧集管理页时，首屏不请求全量分镜和制作资源；制作资源在对应资源 tab 打开后再加载。
- 左侧导航数量和完成态在未打开对应懒加载标签页时仍与后端摘要一致。
- 左侧导航主项与顶部工作台 tab 的顺序和含义一致；点击主项后切换 tab 并回到页面顶部，能看到项目整体设置区。
- 未进入分镜 tab 时，左侧分镜列表仍可从摘要轻量目录展示；点击某条分镜后加载分镜 tab 并跳转到对应卡片。
- 修改项目整体设置后，保存成功时弹出“修改已实时保存，再次生成素材、分镜或视频时将按当前设置生效。”。
- 刷新页面、进入项目或摘要回填项目设置时，不弹出实时保存提示。
- 普通用户无法访问他人项目。
