# 模型用量统计与额度治理

最近更新：2026-06-09

## 功能定位

模型用量统计与额度治理是 AI 短视频平台的成本和稳定性底座，负责记录每次模型调用的真实消耗、估算消耗、额度占用、成本归属和超限处理。

这不是全局并发策略的一部分。全局路由策略负责“怎么跑”，额度治理负责“还能不能跑、跑了算谁的、用了多少”。

## 文档边界

- `02---AI模型配置与全局路由.md`：负责模型配置、服务商协议、路由顺序、失败切换、冷却和运行策略。
- `10---任务队列状态与成本控制.md`：负责任务状态机、并发执行、取消、暂停、重试、进度和失败恢复。
- 本文：负责模型调用后的用量流水、额度策略、周期统计、成本估算、超额动作和告警。

任务队列可以引用本文的额度检查结果，但不在任务文档中重复定义 token、图片张数、视频秒数等计量规则。

## 适用范围

- 文本模型：故事生成、剧本生成、角色/场景/道具提取、分镜生成、提示词润色。
- 图片模型：角色图、场景图、道具图、分镜图、首尾帧、参考图重绘。
- 视频模型：图生视频、文生视频、首尾帧视频、批量分镜视频。
- 音频模型：TTS、旁白、对白、音频生成。
- 文件和存储：后续可扩展存储容量、文件数、导出次数统计。

## 计量维度

额度和统计必须支持多维组合：

- 服务类型：`text`、`image`、`video`、`audio`。
- API 配置：单个 `ai_service_configs.id`，这是当前 AI 配置列表行级限额的默认粒度。
- 配置内模型名称：如 `gpt-image-2`、`seedream`、`veo`、`deepseek`；当同一 API 配置包含多个模型时，可用 `config_id + model` 做细分限额。
- 业务场景：故事、提取、分镜、角色图、分镜图、视频、TTS。
- 用户：创建人、操作人、管理员。
- 项目：单个项目或项目组。
- 周期：日、月、季度、年、自定义周期。

## API 配置行级限额口径

单个 API 配置/模型限额不属于全局路由策略。全局策略只负责路由、并发、重试、超时和冷却；每日限额放在 AI 配置的模型列表行中维护。

当前口径：

- 每条 AI 配置行可设置每日限额，默认 `0` 表示不限制。
- 限额可以作用于整条配置，也可以选择该配置内的某个模型单独设置。
- 文本配置首期支持每日请求数、每日 token。
- 图片配置首期支持每日请求数、每日图片张数。
- 视频配置首期支持每日请求数、每日视频秒数。
- 音频/TTS 配置首期支持每日请求数、每日音频秒数。
- 超额动作支持“自动切换/跳过该配置”“仅提醒，不阻断”“禁用当日”“管理员确认”。当前运行时除 `warn_only` 外都会阻断当前配置继续入选；禁用当日和管理员确认的独立审批流仍属于后续增强。

数据库映射：

```text
scope_type = config
scope_id = ai_service_configs.id
service_type = 当前配置类型
model = 空字符串表示整条配置；非空表示配置内指定模型
period_type = day
limit_unit = requests / tokens / images / video_seconds / audio_seconds
limit_value = 0 表示不限制
action_on_exceed = fallback / warn_only / disable_today / require_admin
```

## 计量单位

| 类型 | 主要单位 | 辅助单位 |
| --- | --- | --- |
| 文本 | `input_tokens`、`output_tokens`、`total_tokens` | 请求数、字符数、结构化输出次数 |
| 图片 | `image_count` | 分辨率、参考图数量、输出格式、抽卡批次 |
| 视频 | `duration_seconds`、`video_count` | 分辨率、模式、首尾帧、任务数 |
| 音频 | `character_count`、`audio_seconds` | 语音角色、语言、任务数 |
| 存储 | `storage_bytes`、`file_count` | 缩略图、缓存、导出文件 |

真实 usage 优先，估算 usage 兜底。服务商返回 token 或扣费字段时记录真实值；不返回时按本地模型价格和参数估算。

## 执行链路

```text
业务入口
  -> ai-runtime 选择候选配置
  -> quotaGuard 检查额度，已超额且非 warn_only 的配置从候选中移除
  -> quotaGuard 可选预占额度（后续任务队列阶段）
  -> adapter 调用模型
  -> usageTracker 记录真实或估算 usage
  -> quotaGuard 结算额度
  -> 失败时释放预占，或记录服务商已扣费的失败消耗
```

## 前端页面规划

AI 配置后续应拆出以下视图：

- 模型用量总览：按日、月、季度查看请求数、token、图片张数、视频秒数、成本；当前已先落地“运行观测”页签，覆盖请求数、失败数、耗时、token 和图片数基础摘要。
- 单配置用量：查看某个 API Key 或中转配置的额度、剩余、失败率。
- 单模型用量：查看某个模型在不同业务场景下的消耗。
- 额度策略：配置每个模型、配置、用户或项目的限额。
- 告警与超限：接近限额、已超限、异常消耗、自动停用记录。

## 当前项目情况

- 当前已有 AI 配置表、全局路由策略、失败冷却和重试基础。
- 当前已落地轻量 `ai_model_call_logs`，用于记录脱敏调用摘要、prompt hash/长度、usage 原始摘要、错误摘要和诊断摘要。
- 当前已落地正式 `ai_usage_events` 用量事件表；模型调用日志写入成功后会同步写一条 usage event，作为后续额度、周期和成本统计的账本基础。
- 当前已通过 `modelRuntimeService` 和 AI 配置“运行观测”页签提供基础只读统计：按服务类型、状态、时间范围查看汇总和最近调用。
- 当前新增 `GET /ai-configs/model-usage-daily`，可按日期、服务类型、配置、模型和业务场景查询正式用量日汇总。
- 当前已新增 `ai_quota_policies`、`ai_quota_counters` 和额度策略基础接口：`GET/POST/PUT/DELETE /ai-configs/quota-policies`。
- 当前运行时配置候选会执行额度检查：超出策略且动作不是 `warn_only` 的配置会被跳过，从而触发后续候选兜底；预检已接入文本 token、图片张数、视频秒数和音频秒数估算。
- 当前 AI 配置列表已增加行级“今日用量 / 每日限额”入口，管理员可在配置行上直接维护单配置或配置内模型的日限额，并设置超额动作。
- 当前已新增登录用户可读的 `GET /runtime/model-usage-daily`，并在首页和创作页顶部提供“用量中心”入口，查看整站文本、图片、视频、音频用量。
- 当前任务详情接口会返回关联模型调用诊断摘要，可按 `task_id` 追踪 trace、错误、耗时和用量。
- 当前尚未正式落地成本估算、额度预占释放和用户级个性化限额页面。
- 本规范先建立目标边界；后续做模型模块时继续补全价格规则、任务预占释放和页面。

## 待开发清单

- 继续补齐更多入口的 `user_id` 和 adapter usage 来源，避免部分后台链路只记录项目或任务维度。
- 视频和音频当前先以请求参数估算秒数入账；后续应结合服务商返回或文件探测结果修正为真实完成消耗。
- 建立模型价格规则迁移和成本估算。
- 将额度预占、释放和结算接入长耗时任务队列，避免并发任务同时越过额度线。
- 用量中心后续补用户维度、个人明细、成本、告警和导出能力。
- 任务详情后续在前端显式展示本次模型调用的耗时、消耗和额度归属。

## 验收清单

- 单个模型、单个 API 配置、单个周期可以限制用量。
- 超额后能按策略阻止、切换、冷却或仅提醒。
- 文本 token、图片张数、视频秒数、音频字符都能被记录。
- 失败、取消、超时任务能区分是否已产生模型消耗。
- 普通用户不能看到其他用户的成本和敏感配置。
