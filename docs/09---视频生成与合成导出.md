# 视频生成与合成导出

最近更新：2026-06-10

## 功能定位

视频生成把分镜图、分镜文本、首尾帧、参考图和模型配置转成可播放视频。合成导出把多个分镜视频按分集顺序合成为完整成片。

## 当前项目情况

- 视频生成记录落在 `video_generations`，包含项目、分镜、模型、AI 配置、时长、画幅、主参考图、首帧、尾帧、多参考图、视频 URL、本地路径、任务和实际参数。
- 当前支持单条视频生成、由图片生成视频、按分集批量生成视频、查询视频、删除视频。
- 当前支持整集合成，合成记录落在 `video_merges`。
- 视频客户端已适配多个服务商协议，复杂度明显高于文本和图片。
- 批量生成视频时会优先处理有参考图且没有已完成视频的分镜，并记录失败项。
- 当前最需要补的是生成前参数摘要、失败重试策略和合成前完整性检查。

## 当前已落地能力

- 支持从图片生成视频。
- 支持按分集批量生成分镜视频。
- 支持全能模式下多参考图或片段描述生成视频。
- 支持连贯帧：用上一条视频末帧辅助下一条视频。
- 支持视频生成任务记录、轮询、错误状态。
- 支持整集视频合成。

## 当前问题与不足

- 不同服务商视频能力差异大，参数契约需要更清楚。
- 全能模式、图生视频、首尾帧和连贯帧之间的组合规则需要更稳定。
- 批量生成和合成前缺少完整风险检查。
- 视频失败后的重试、换模型、跳过、保留旧视频策略需要明确。

## 方案池 / 待确认讨论

- 视频生成前显示“本次使用：模型、模式、时长、画幅、参考图数量、首尾帧状态”。
- 合成前增加分镜视频完整性检查。
- 支持按失败原因推荐操作：换模型、减少参考图、缩短时长、重建提示词。

## 待开发清单

- 梳理各视频服务商能力矩阵。
- 明确视频生成参数优先级。
- 给合成导出增加缺失分镜提示。

## 验收清单

- 分镜视频生成前，用户能看到完整参数摘要。
- 批量生成失败后，不影响已成功的视频记录。
- 合成导出只使用用户明确选定或最新可用的视频片段。
